# Comparaison des structures de données

| Structure | Avantages | Inconvénients |
| --- | --- | --- |
| Liste (`list`) | Simple à utiliser <br/> Accès rapide par index $O(1)$ <br/> Très flexible | Recherche lente $O(n)$ <br/> Insertion/suppression au milieu coûteuse $O(n)$ |
| Pile (LIFO) | Opérations `push`/`pop` en $O(1)$ <br/> Idéale pour l'historique et la récursion | Accès limité au sommet <br/> Pas de recherche efficace |
| File (FIFO) | Ajout/retrait en $O(1)$ avec `deque` <br/> Utile pour les files d'attente | Accès limité aux extrémités <br/> Pas d'accès direct à un élément interne |
| Dictionnaire (`dict`) | Accès, insertion et suppression en $O(1)$ en moyenne <br/> Structure clé-valeur très pratique | Coût mémoire plus élevé <br/> Non ordonné conceptuellement |
| Arbre binaire de recherche (ABR) | Données ordonnées <br/> Recherche, insertion et suppression en $O(\log n)$ en moyenne | Peut se déséquilibrer (pire cas $O(n)$) <br/> Implémentation plus complexe |

## Résumé rapide

- **Liste** : collections générales avec accès par index.
- **Pile** : traitement en ordre dernier entré, premier sorti (LIFO).
- **File** : traitement en ordre premier entré, premier sorti (FIFO).
- **Dictionnaire** : retrouver rapidement une valeur à partir d'une clé.
- **ABR** : données ordonnées avec beaucoup d'opérations de recherche, d'insertion et de suppression.
